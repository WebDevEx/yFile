Library Files

The yFiles for HTML library consists of the content of the lib directory, namely the JavaScript library modules in UMD format in lib/umd, the same functionality in ES6 module library format in lib/es-modules, and the stylesheet lib/es-modules/yfiles.css (or lib/umd/yfiles.css) that is required for both of the former options.

In order to develop applications with yFiles for HTML, you need to provide valid license information. Please see the related section in the Developer's Guide for the necessary setup. Evaluation versions of yFiles for HTML come with a pre-generated evaluation license file that is located in lib/license.json. You can include this file in any HTML page that uses the library.

For detailed descriptions see also the Setting up your Development Environment section of the yFiles for HTML Developer's Guide.

The top-level directory deployment contains various examples that demonstrate the transpilation, optimization, and minification of yFiles for HTML applications. For more details, see the description in the deployment directory or the respective section in the Developer's Guide.

Working with the yFiles npm Module

Since yFiles for HTML 2.2, the library comes as an npm module. This makes it easy to install it into your projects.

The easiest way to install the yFiles for HTML npm package is using a local file path. Once you have extracted the yFiles package and run the setup script (npm install or one of the start_demos scripts), you will have four npm packages to choose from:

  • lib/es-modules/yfiles-<version>.tgz
  • lib-dev/es-modules/yfiles-<version>-dev.tgz
  • lib/umd/yfiles-<version>.tgz
  • lib-dev/umd/yfiles-<version>-dev.tgz

It is recommended to always depend on the *.tgz files when using yFiles for HTML as npm dependency. For more information, see the dedicated section in the Developer's Guide.

yFiles Developer Tools

Yeoman Generator

The Yeoman generator for yFiles can be used to quickly scaffold a yFiles for HTML application. See its GitHub page for a more detailed description.

IDE Code Completion

Auto-completion support in your IDE can be enabled through one of the typings files in ide-support/. Please see the Setting up Your Development Environment for more information about the different IDEs and typings files.

TypeScript

TypeScript support in your IDE can be enabled through one of the various ide-support/yfiles-api-*.d.ts file variants.

Runtime Type Checking

During development of your application, you can get runtime type information for library types and parameter lists of methods in your browser's JavaScript console by including the following JavaScript file: ide-support/yfiles-typeinfo.js

For detailed descriptions see also the yFiles Developer Tools section of the yFiles for HTML Developer's Guide.

Code Conversion Tools

The tools directory contains several scripts to convert your source code between various ways to import the yFiles library, e.g. from UMD to ES module imports.

Unobfuscated Library

In the debug package of yFiles for HTML, the lib.debug directory contains an unobfuscated version of the library. These files are not redistributable. They may be used for debugging purposes, only.
